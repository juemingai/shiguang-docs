---
title: 策划编剧（短剧）
description: POST /api/agent/planning
---

根据故事创意，生成面向抖音、快手平台的短剧结构化策划方案，包含题材标签、故事大纲、爽点设计、人物小传和分集梗概。

支持多轮修改。

---

## 请求

**`POST /api/agent/planning`**

### 请求体

<ParamField body="user_input" type="string" required>
  故事创意描述，一句话即可。

  示例：`"一个古代穿越现代的甜宠故事"`
</ParamField>

<ParamField body="episode_count" type="integer" default={10}>
  集数，范围 1 ~ 100。
</ParamField>

<ParamField body="feedback" type="string">
  补充说明或修改意见。首次生成可传附加要求，多轮修改时传修改意见。
</ParamField>

<ParamField body="previous_response_id" type="string">
  多轮修改时传入上一次 `done` 事件中的 `response_id`。首次生成不传此字段。
</ParamField>

### 示例请求

<CodeGroup>

```bash 首次生成
curl -X POST http://localhost:8000/api/agent/planning \
  -H "Content-Type: application/json" \
  -N \
  -d '{
    "user_input": "一个古代穿越现代的甜宠故事",
    "episode_count": 12
  }'
```

```bash 多轮修改
curl -X POST http://localhost:8000/api/agent/planning \
  -H "Content-Type: application/json" \
  -N \
  -d '{
    "user_input": "一个古代穿越现代的甜宠故事",
    "episode_count": 12,
    "feedback": "男主设定改为现代 CEO，增加商战元素",
    "previous_response_id": "resp_0217xxxx"
  }'
```

```python Python
import httpx

with httpx.stream("POST", "http://localhost:8000/api/agent/planning", json={
    "user_input": "一个古代穿越现代的甜宠故事",
    "episode_count": 12,
}) as r:
    for line in r.iter_lines():
        if line.startswith("data: "):
            print(line[6:])
```

</CodeGroup>

## 响应

**`200 OK`** — SSE 流式响应（`text/event-stream`）

### chunk 事件

```
data: {"chunk": "{\"genre_tags\": [\"古装\", \"甜宠\"]]"}

data: {"chunk": ", \"premise\": \"落魄千金穿越古代，邂逅冷峻王爷\"..."}
```

生成完成后推送一次，包含完整结构化策划数据。

### planning 字段结构

<ResponseField name="genre_tags" type="string[]">
  题材标签，如 `["都市", "甜宠", "逆袭"]`
</ResponseField>

<ResponseField name="premise" type="string">
  一句话故事前提
</ResponseField>

<ResponseField name="outline" type="string">
  故事大纲，300 字以内
</ResponseField>

<ResponseField name="hook_points" type="string[]">
  核心爽点列表
</ResponseField>

<ResponseField name="twists" type="string[]">
  关键反转列表
</ResponseField>

<ResponseField name="rhythm_notes" type="string">
  节奏设计说明
</ResponseField>

<ResponseField name="characters" type="object[]">
  主要人物列表

  <Expandable title="角色字段">
    <ResponseField name="name" type="string">角色姓名</ResponseField>
    <ResponseField name="role" type="string">角色定位（如"女主"）</ResponseField>
    <ResponseField name="age" type="string">年龄</ResponseField>
    <ResponseField name="appearance" type="string">外貌描述</ResponseField>
    <ResponseField name="personality" type="string">性格特征</ResponseField>
    <ResponseField name="background" type="string">人物背景</ResponseField>
    <ResponseField name="motivation" type="string">核心驱动力</ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="episodes" type="object[]">
  分集梗概列表

  <Expandable title="分集字段">
    <ResponseField name="episode_number" type="integer">集数</ResponseField>
    <ResponseField name="title" type="string">本集标题</ResponseField>
    <ResponseField name="summary" type="string">本集剧情梗概</ResponseField>
    <ResponseField name="hook" type="string">本集结尾钩子</ResponseField>
    <ResponseField name="key_scene" type="string">核心场景描述</ResponseField>
  </Expandable>
</ResponseField>
